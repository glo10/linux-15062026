# Atelier 1.2 : démarrage et arrêt du système

## Objectifs

- [x] Maîtriser les commandes d'arrêt planifié
- [x] Avec VirtualBox, basculer entre les consoles virtuelles
- [x] Modifier le comportement de démarrage du système

---

## Partie I : Planification arrêt du système

1. Démarrez la VM crée précédemment dans l'atelier 1.1
2. Ouvrez un deuxième terminal virtuelle en utilisant le raccourci CTRL + ALT + F2 ou CTRL DROIT + F2 et connectez-vous pour lancer une deuxième session.
3. Depuis ce deuxième terminal, planifiez l'arrêt du système dans 10 minutes avec le message suivant à diffuser "Maintenance, arrêt dans 10 minutes"
- Il est possible et normal qu'à la suite de l'exécution de votre commande que le terminal reste bloqué
4. Basculez sur la première session avec CTRL + ALT + F1 pour voir le message broadcast indiquant l'arrêt du système dans 10 minutes.
5. Basculez dans la deuxième session avec CTRL + ALT + F2, annulez l'arrêt planifié et planifiez l'arrêt du système à 23h30 avec le message "Maintenance prévue aujourd'hui à 23h30".

---

## Partie II : mode de démarrage

6. Identifiez le mode de démarrage par défaut à l'aide de la commande suivante
```bash
systemctl get-default
```
7. Sans redémarrer la machine, basculez en mode "Secours" à l'aide la commande suivante et suivez les instructions du système
```bash
sudo systemctl isolate rescue.target # ou sudo systemctl rescue
```
8. Exécutez les commandes suivantes pour savoir sur quel mode vous êtes
```bash
# 1 ou S ou unknow => mode rescue
sudo runlevel 
# running ou degraded => mode normal, maintenace => mode rescue
systemctl is-system-running
```
9. Rebasculez en mode normal sans interface graphique à l'aide de la commande suivante en exécutant l'une des commandes suivantes
```bash
systemctl isolate multi-user.target
# ou
systemctl default
```
10. Eteignez la machine et effectuez un snapshot depuis VirtualBox nommé ***v1.2***.
