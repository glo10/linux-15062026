# Correction Atelier 3.1 : commandes GNU et UNIX

## Objectifs

- [x] Matrisez la création des répertoires et fichiers
- [x] Maitrisez le téléchargement des ressources extérieurs depuis wget


## Recap commandes

|                                  | Commande                |
| -------------------------------------- | ----------------------- |
| Créer un répertoire                    | `mkdir`                 |
| Créer plusieurs répertoires            | `mkdir -p`              |
| Créer un fichier vide                  | `touch fichier`         |
| Éditer un fichier                      | `nano fichier`          |
| Afficher le contenu d'un dossier       | `ls`                    |
| Afficher récursivement                 | `ls -R`                 |
| Afficher les fichiers cachés           | `ls -a`                 |
| Affichage détaillé                     | `ls -lah`               |
| Télécharger un fichier                 | `wget URL`              |
| Télécharger dans un dossier spécifique | `wget -P <dossier> URL` |
| Identifier le type d'un fichier        | `file fichier`          |
| Créer un alias temporaire              | `alias nom='commande'`  |
| Recharger `.bashrc`                    | `source ~/.bashrc`      |

## Partie I : manipulation des fichiers et dossiers internes

1. Création des dossiers
```bash
# Brace Expansion
mkdir -p ~/projets/{web,mobile} ~/{pdf,images,docs}
```
2. Création des fichiers avec nano  :
```bash
cd ~/projets/web
touch README.md index.html .env
# ou bien
touch ~/projets/web/{README.md,index.html,.env}
```
3. Ajout du contenu html dans index.html
```bash
nano ~/projets/web/index.html
```
4. Ajout contenu Markdown dans README.md.

```bash
nano ~/projets/web/README.md
```

----

## Partie II : : manipulation des fichiers externes et cachés

5. Affichage recursive du répertoire personnel ~/.
```bash
tree ~
ls -R ~
# . représente le répertoire courant .. le répertoire parent
```
6. Téléchargement des images
```bash
wget -P ~/images/linux.png https://cognitivewaves.wordpress.com/wp-content/uploads/2017/03/gnu-linux-distribution-timeline.png
wget -P ~/images/ubuntu.webp https://pic.clubic.com/be434f791908768/1600x1067/smart/ubuntu-1.webp
wget -P ~/images/debian.png https://www.debian.org/Pics/debian-logo-1024x576.png
wget -P ~/images/fedora.png https://i0.wp.com/goodtech.info/wp-content/uploads/2025/02/fedora.png
```
7. Liste dossiers et fichiers dans  ~/projets/web et conclusions
```bash
ls ~/projets/web
```
- Les fichiers cachés (commençant par .) n'apparaissent pas, ici .env n'apparaît pas dans le résultat.
8. Option -a pour voir tous les fichiers même ceux qui sont cachés
```bash
ls -lah ~/projets/web
```
9. Type des fichiers
```bash
file ~/projets/web/*
```
10. Explication sur le résultat de la commande suivante
```bash
file ~/projets/web/*
```
- Idem .env n'apparaît pas dans les résultats
- * ne tient pas comptes des fichiers cachés
- file ne se base pas sur l'extension mais sur le contenu présent dans le fichier
```bash
# Pour le faire apparaître
file ~/projets/web/{*,.*}
# ou bien
file .env * 
```
