## Recap commandes

| ------------------------- | ------------------------------ |

| Processus en arrière-plan | `commande &`                   |
| Lister les jobs           | `jobs`                         |

| Afficher quelques colonnes          | `ps -o pid,ni,stat`            |


## Signaux

| Signal    | Numéro | Résultat                                    |
| --------- | :----: | ---------------------------------------- |
| `SIGHUP`  |    1   | Recharger la configuration               |
| `SIGINT`  |    2   | Interruption (`Ctrl+C`)                  |
| `SIGQUIT` |    3   | Quitter en vidant la mémoire              |
| `SIGTERM` |   15   | Arrêter propre proprement (par défaut de `kill`)      |
| `SIGSTOP` |   19   | Suspension immédiate (non interceptable) |
| `SIGTSTP` |   20   | Suspension depuis le terminal (`Ctrl+Z`) |
| `SIGCONT` |   18   | Reprise d'un processus suspendu          |
| `SIGKILL` |    9   | Arrêt forcé (non interceptable)          |

9. Vérification inexistance du processus sleep avec le même PID

10. Tmux
```bash

while true; do :; done & # depuis l'autre fénêtre
```
- 2 fenêtres horizontales : CTRL+B puis "
- Navigation entre les 2 fenêtres : CTRL+B puis ↑ ou ↓
- Pour quitter : CTRL+B puis d
- Le processus (boucle infinie) consomme pratiquement 100 % d'un cœur CPU.
11. Lancement des processus
```bash
sudo apt install tmux
tmux new
htop # depuis une fénêtre
while true; do :; done & # depuis l'autre fénêtre
```
12. Baisser sa priorité et conclusions sur la conso CPU
```bash
ps -C bash -o pid,cmd
# ou bien
jobs -1
renice 15 -p <PID>
# ou bien
renice -n 15 <PID>
```
- Conclusions :
    - On constate que la consommation du CPU ne baisse pas vraiment car il n'y a pas beaucoup de processus qui utilisent le CPU donc la boucle infinie continue à utiliser toutes les ressource. Par contre, elle cèdera sa place dès qu'un autre processus demandera du calcul.
    - S'il y avait beaucoup de processus qui demandent le CPU, la boucle infinie consommerait beaucoup moins de CPU car les autres processus sont plus prioritaires avec un nice à 0 par défaut.
13. Fin processus boucle infinie
```bash

# ou bien via le job

```

16. Processus zombie
```python
python3 -c "import os, sys, time; pid = os.fork(); sys.exit() if pid == 0 else time.sleep(60)" &
```
17. Trouvez ce processus zombie dans votre liste de processus. Quel est son état (la lettre dans la colonne STAT) ? Pouvez-vous le tuer avec un kill -9 ?
```bash
ps -aux -o ppid,pid,stat | grep Z
kill -9 <PID> # ne sert à rien
```
- Le processus enfant est déjà mort, il a un état Z pour Zombie, un kill n'aura aucun effet.
18. Terminez-le proprement, puis vérifiez qu'il n'existe plus.
```bash
ps -o pid,ppid,state,cmd -p <PID>
# ou bien
ps -o ppid | grep Z
kill <PPID>
# Ou bien à partir du job
kill %<JOB>
```
- On ne peut pas tuer un zombie (kill -9 ne fonctionne pas) car il est déjà mort (processus déjà terminé).
- La seule façon de le faire disparaître est de tuer son processus parent ou d'attendre que le parent se termine.
- Ne consomme pratiquement aucune ressource (ni CPU, ni mémoire utilisateur)
- Disparaît uniquement lorsque son processus parent appelle *wait()* ou se termine.
