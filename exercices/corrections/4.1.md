## Enoncé

1. Ajout nouveau disque.
- VirtualBox → Configuration → Stockage
- Ajouter un disque dur virtuel :
    - Taille : 3 Go
    - Format : VDI
    - Allocation dynamique
    - Contrôleur SATA (ou SCSI)
2. Démarrez et repérez le nouveau disque.
```bash
lsblk
```
Le disque devrait logiquement s'appeler ***/dev/sdb***
3. Partition de 1Go
```bash
gdisk /dev/sdb
# n nouvelle partition
# p primaire
# 1 numéro
# Entrée  premier secteur
# Taille +1G
# w enregistrer
```
4. Formatage et montage automatique au démarrage
- Formatage
```bash
mkfs.ext4 /dev/sdb1
```
- Montage dans `/mnt/tutos`
```bash
mkdir -p /mnt/tutos
mount /dev/sdb1 /mnt/tutos
blkid /dev/sdb1 # récupérer le UUID
# vi /etc/fstab pour ajouter UUID=7a9f...   /mnt/tutos   ext4   defaults   0   2
# Vérifications
umount /mnt/tutos
mount -a
df -h 
```
5. Voir swap disponible
```bash
free -m
```
6. Partition de 200Mo en plus pour le swap
```bash
gdisk /dev/sdb
# n
# p
# 2
mkswap /dev/sdb2
swapon /dev/sdb2
# Pour rendre le swap permanent (au démarrage)
blkid /dev/sdb2
# vi /etc/fstab et ajoutez UUID=<UUID de /dev/sdb2> none swap sw 0 0
# Vérifications
swapoff /dev/sdb2
swapon -a
swapon --show
```
7. Partition de 1.8Go type ext4 montée sur `/mnt/multimedia`
```bash
gdisk /dev/sdb
# n
# p
# 3
# 2 fois Entrée
# Formatage
mkfs.ext4 /dev/sdb3
# Montage
mkdir -p /mnt/multimedia
mount /dev/sdb3 /mnt/multimedia
# Montage automatique au démarrage
blkid /dev/sdb3
# vi /etc/fstab ajoutez UUID=<UUID de /dev/sdb3> /mnt/multimedia ext4 defaults 0 2
umount /mnt/multimedia
mount -a
```
- Partition occupe toute l'espace restante
8. Vérification intégrité
```bash
# Prendre des précautions avant les vérifs avec le démontage des partitions
umount /mnt/multimedia
umount /mnt/tutos
e2fsck -f /dev/sdb1
e2fsck -f /dev/sdb3
```
- -f pour forcer la vérification du FS même lorsque ce dernier est marqué *clean*
- Le swap est un cas particulier on ne peut pas vérifier

9. Quota sur /mnt/multimedia
```bash
# 1. Démontage
sudo umount /mnt/multimedia
# 2. Activation de la fonctionnalité des quotas sur le système de fichiers ext4 (nativement intégré mais désactivé)
sudo tune2fs -O quota /dev/sdb3
# 3. Vérification
sudo tune2fs -l /dev/sdb3 | grep "Filesystem features" # affichage quota
# 4. Définition du type de quotas
sudo tune2fs -Q usrquota,grpquota /dev/sdb3
# 5. Édition de /etc/fstab pour ajouter les options de quota usrquota et grpquota
sudo vi /etc/fstab
#   UUID=xxxxxxxx /mnt/multimedia ext4 defaults,usrquota,grpquota 0 2
# 6. Remontage
sudo mount -a
# 7. Activation des quotas sur la partition
sudo quotaon /mnt/multimedia
```
10. Limitation à 50Mo soft et 100Mo stricte
```bash
sudo edquota -u $(whoami) # mode interactif
# ou bien en mode manuel 50Mo = 51200 Ko et 100Mo = 102400 Ko
sudo setquota -u $(whoami) 51200 102400 0 0 /mnt/multimedia
# Vérification quota utilisateur courant
quota -u $(whoami)
# Test création des fichiers pour vérifier les limites
##  Création fichier de 60Mo pour dépasser la limite soft
dd if=/dev/zero of=/mnt/multimedia/test.img bs=1M count=60
##  Création fichier de 50Mo en plus de la précédente pour dépasser la limite stricte
dd if=/dev/zero of=/mnt/multimedia/test2.img bs=1M count=50
```
- $(whoami) retourne l'utilisateur courant
- Dans les tests, la création du fichier /mnt/multimedia/test2.img échoue car dépassement de la limite de 100Mo
- Limite soft  = seuil d'avertissement, dépassement temporaire autorisé durant une période qui est configurable avec la commande `edquota -t`
- Limite hard = limite absolue

11. Démontage

```bash
umount /mnt/multimedia
```
12. Vérifications
```bash
e2fsck -f /dev/sdb1
e2fsck -f /dev/sdb3
```
13. 
- Suppression et création des nouvelles partitions, attention, ici, il faut recréer les nouvelles partitions avec les nouvelles tailles sans modifier leur secteur de départ (laisser la valeur par défaut proposé par gdisk)
```bash
 gdisk /dev/sdb
# d puis 2 entrée => suppression de la partition 2 correspondant au swap
# d 1 suppression partition 1
# d 3 suppression partition 2
# n création partition 1
# n création partition 2
# w
```
- Agrandissement
```bash
resize2fs /dev/sdb1
resize2fs /dev/sdb3
# Vérification
df -h
```

14. Redémarrage
```bash
reboot
```
15. Constats sur le redimentionnement :
- Avant de redimentionner, il faut s'assurer que :
    - **Effectuer une sauvegarde de votre système de fichiers**, en effet, une erreur peut entrainer une perte totale des données stockées.
    - **la partition est demontée avec umount**
    - **la partition est intègre (e2fsck ne renvoit pas d'erreur)**
- Durant le redimentionner, il faut adapter à la fois la table des partitions et le système de fichiers.
- Agrandir une partition est généralement plus simple que la réduire.
