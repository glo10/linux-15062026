## Synthèse commandes

|                                            | Commande                                    |
| ------------------------------------------ | ------------------------------------------- |
| Installer un paquet sous Fedora            | `sudo dnf install <paquet>`                 |
| Installer un paquet sous Debian            | `sudo apt install <paquet>`                 |
| Lister les fichiers d'un paquet            | `dpkg -L <paquet>`                          |
| Afficher les dépendances dynamiques        | `ldd <exécutable>`                          |
| Activer et démarrer un service             | `sudo systemctl enable --now <service>`     |
| Télécharger un fichier                     | `wget -P <répertoire> <URL>`                |
| Extraire une archive `.tar.xz`             | `tar -xJvf archive.tar.xz -C <destination>` |
| Installer un paquet `.deb` local           | `sudo dpkg -i fichier.deb`                  |
| Corriger les dépendances manquantes        | `sudo apt --fix-broken install`             |
| Afficher la variable `PATH`                | `echo $PATH`                                |
| Ajouter temporairement un chemin au `PATH` | `export PATH=<chemin>:$PATH`                |

2. Installation paquets nano et docker

```bash
dnf install nano docker
# Vérifications des installations

```
3. Installation okular via DNF, observez le retour de votre commande d'installation, que pouvez-vous en déduire ?
```bash
dnf okular
```
- Okular a besoins des dépendances pour fonctionner
- Le gestionnaire de paquet DNF propose de télécharger directement les dépendances nécessaires avant d'installer Okular
- DNF gère automatiquement les dépendances
- Les dépendances graphiques ont beaucoup de sous-dépendances donc lourds

```bash
sudo docker ps
```
- Aucun processus en cours, donc le service docker n'est pas démarré. Cela dépend du système d'initialisation parfois un paquet est téléchargé puis démarré immédiatemment.
5. Etat et lancement du service docker avec systemctl
```bash
sudo systemctl status docker.service
sudo systemctl enable --now docker.service
# Vérification
sudo docker ps # affiche un tableau vide de la liste des conteneurs
```

## Partie II : gestionnaires de paquetage `apt` et `dkpg`

7. Installez nano et docker.io via apt
```bash
sudo apt-get install nano docker.io
# vérifs
nano --version
docker --version
```
8. Utilisez la commande dpkg -L nano pour localiser le fichier exécutable de nano (dans /bin ou /usr/bin).
```bash
dkpg -L nano
```
- Le fichier exécutable est ***`/usr/bin/nano`*** ou ***`/bin/nano`***
9. Exécutez la commande `ldd` sur l'exécutable de nano. Combien de bibliothèques partagées (.so) ce programme a-t-il besoin ?
```bash
ldd /usr/bin/nano
# Sortie
linux-vdso.so.1
libncursesw.so.6 => /lib/x86_64-linux-gnu/libncursesw.so.6
libtinfo.so.6 => /lib/x86_64-linux-gnu/libtinfo.so.6
libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6
/lib64/ld-linux-x86-64.so.2
```
- Normalement il y aura entre 4 et 6 bibliothèques partagées

## Partie III : gestion d'une archive

10. Téléchargement de Node avec wget [Téléchargez l'archive de Node](https://nodejs.org/dist/v26.3.0/node-v26.3.0-linux-x64.tar.xz) au format .tar.xz.
```bash
wget https://nodejs.org/dist/v26.3.0/node-v26.3.0-linux-x64.tar.xz /tmp
```
11. Extrayez l'archive dans /tmp à l'aide de la commande ***tar -cvzf node-v26.3.0-linux-x64.tar.xz /tmp ***
12. Testez son exécution à l'aide de la commande
```bash
/tmp/node.exe --version
```
