# Partie I : droits et permissions

1. Listez tous les utilisateurs
```bash
cat /etc/passwd # chaque correspond un utilisateur
```
2. Création des utilisateurs web et mobile
```bash
sudo useradd web
sudo useradd mobile
# Définition des mots de passe
sudo passwd web
sudo passwd mobile
```
3. UID et GID de web
```bash
id web
```
4. Création groupe projets
```bash
sudo groupadd projets
```
5. Création arboresnce
```bash
mkdir -p ~/projets/{web,mobile}
```
6. Propriétaire et permissions sur le dossier `~/projets`
```bash
sudo chown -R :projets ~/projets
chmod 775 ~/projets
```
7. Propriétaire et permission sur le dossier `~/projets/web` de manière récursive
```bash
sudo chown -R :web ~/projets/web
```
8. Propriétaire et permission sur le dossier `~/projets/mobile` de manière récursive
```bash
sudo chown -R :mobile ~/projets/mobile
```
9. Création dossier `/share`
```bash
mkdir /share
sudo chown -R root:projets /share
sudo chmod 2775 /share # SETGUID => 2 tous les fichiers héritent du groupe projets
```
## Partie II : liens physiques et symboliques

10. Groupes rh et compta
```bash
sudo groupadd rh
sudo groupadd compta
```
11. Création répertoire `/opt/budgets`
```bash
sudo mkdir -p /opt/budgets
```
12. Propriétaires utilisateur root et groupe compta
```bash
sudo chown root:compta /opt/budgets
```
13. Restrictions et accès uniquement aux membres du groupe compta
```bash
sudo chmod 770 /opt/budgets
```
14. Tentative d'accès par un utilisateur qui n'appartient pas au groupe
```bash
su - web
ls /opt/budgets # accès refusé
```
15.  Création `/opt/budgets/2026.txt` par root
```bash
sudo touch /opt/budgets/2026.txt
ln -s /opt/budgets/2026.txt /home/web/budget_2026.txt
```
16. Lecture du lien symbolique depuis le compte de l'utilisateur web
```bash
su - web
cat ~/budget_2026.txt
```
- Les permissions s'appliquent au fichier cible, pas au lien.
17. Suppression par root du fichier d'origine (cible)
```bash
sudo rm /opt/budgets/2026.txt
ls -l /home/web/budget_2026.txt # le lien est cassé
```

## Bonus

18. Création des utilisateurs
```bash
sudo useradd alice
sudo useradd bob
sudo useradd charlie
```
19. Création desgroupes
```bash
sudo groupadd dev
sudo groupadd ops
# affectations
sudo usermod -aG dev alice
sudo usermod -aG dev bob
sudo usermod -aG ops charlie
```
20. Création dossier /var/www
```bash
sudo mkdir -p /var/www
sudo chown root:dev /var/www
sudo chmod 2770 /var/www
```
- Ici on atteint les limites du standard avec des droits spécifiques sur le groupe ops (dans le standard actuel il ferait parti des autres or on veut qu'il est rx tandis que pour les autres il y a aucun droit)
- Comme on atteint les limites, la solution est d'utiliser ACL
```bash
sudo setfacl -m g:ops:rx /var/www # les droits pour ops
sudo setfacl -m o::--- /var/www # les droits pour les autres
```
21. Dossiers et fichiers dans /var/www appartiennent systématiquement à dev
```bash
sudo chmod g+s /var/www # Normalement ça déjà été fait à la question précédente
```
22. Bob ne peut pas modifier les fichiers crées par alice
```bash
sudo chmod +t /var/www
# ajout masque
umask 007 # droits à la création par défaut 770 et fichiers 660
```
23. 3 shells pour 3 utilisateurs
```bash
tmux new
# CTRL+B " (2 fois pour avoir 3 fénêtres)
su - alice      # 1er fénêtre
su - bob        # 2e fénêtre
su - charlie    # 3e fénêtre
```
24. Vérifications
```bash
# Alice ou Bob
touch /var/www/test.txt
ls -l /var/www
```
- Alice et Bo appartenant au groupe dev : lecture/écriture/ création OK
- Charlie (ops) : lecture
- Autres : accès refusé
