## Partie I

3. Arrêt du système dans 10 minutes
```bash
sudo shutdown -h +10 "Maintenance, arrêt dans 10 minutes"
```
- Le message est diffusé dans toutes les sessions (utilisateurs connectés sur le système). On parle de broadcast (diffusion  du message à tous les utilisateurs connectés)
5. Annulation arrêt système planifié dans 10 min.
```bash
sudo shutdown -c
```
- Broadcast annulation de l'arrêt, si l'un des terminaux était bloqué, à la suite de cette annulation, il se débloque.
- Planification arrêt du système à 23h30
```bash
sudo shutdown -h 23:30 "Maintenance planifiée à 23h30"
```
- Le système reste opérationnel jusqu'à 23h30, sauf si la planification est modifée ou annulée

## Partie III

6. Mode de démarrage par défaut
```bash
systemctl get-default
```
- Normalement c'est le mode ***multi-user.target*** (pour une installation en mode texte) ou ***graphical.target*** (pour une installation graphique)
7. Mode "Secours"
- La majorité des services tels que le réseau et donc les sessions des utilisateurs seront coupées
- Les autres utilisateurs sont déconnectés
- Le système demande le mot de passe root pour continuer
- Les fichiers seront en lecture seul
```bash
systemctl isolate rescue.target # ou bien  systemctl rescue
# 1. touche Entrée
# 2. Se logger avec root
```
8. Mode actuel
- Avec runlevel N 1 ou 1 1 
- Avec systemctl maintenance
9. Retour à la normale
```bash
sudo systemctl isolate multi-user.target # en mode texte
# ou bien
sudo systemctl default
# Vérification
systemctl is-system-running # running
```

10. Arrêt de la machine
```bash
sudo shutdown -h now
# ou bien
sudo poweroff
```
- Effectuez votre 2ème snapshot nommé V1.2

## Bonus

11. Démarrage par défaut mode texte
```bash
systemctl set-default multi-user.target
# PS : pour forcer en mode graphique systemctl set-default graphical.target
# Vérification
systemctl get-default # multi-user.target
```
12. reboot
```bash

# redémarrage en mode texte
# Vérification après redémarrage
systemctl get-default
```
