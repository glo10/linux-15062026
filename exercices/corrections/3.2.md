## Partie I : filtres et redirections

1. Sauvergade de tous les paquets dans ~/paquets.txt
```bash
dpkg -l > ~/paquets.txt   # (Debian)
rpm -qa > ~/paquets.txt   # (Fedora)
```
2. Infos sur les cartes réseaux
```bash
dmesg | grep -E "eth|enp|ens|network"
```
3. A l'affichage remplacement ***Bienvenue*** par ***Welcome*** depuis le fichier ~/projets/web/index.html
```bash
cat ~/projets/web/index.html | tr "Bienvenue" "Welcome"
# ou bien avec sed qu'on n'a pas vu
sed 's/Bienvenue/Welcome/' ~/projets/web/index.html
```
- sed est un éditeur de flux (stream editor).
    - s/ancien/nouveau/ signifie "substituer la première occurrence de ancien par nouveau" attention au 3 / à ne pas oublier ou mal placer.
    - Le fichier d'origine n'est pas modifié, seule la sortie affichée est transformée.
- tr convertir, compresser ou éliminer des caractères de l'entrée standard en écrivant sur la sortie standard
4. Affichage précédent redirigé vers le fichier ~/tmp
```bash
mkdir ~/tmp
sed 's/Bienvenue/Welcome/' ~/projets/web/index.html > ~/tmp/index.html
# ou bien
cat ~/projets/web/index.html | tr "Bienvenue" "Welcome" > ~/tmp/index.html
```
5. Enchaînement de commandes pour afficher les utilisateurs possedant bash depuis le fichier /etc/passwd
```bash
grep '/bin/bash$' /etc/passwd | wc -l
```
- /etc/passwd contient la liste de tous les utilisateurs du système. Chaque ligne se termine par le shell de connexion par défaut
- grep "/bin/bash$" sélectionne les lignes se terminant par /bin/bash.
- | transmet ces lignes à wc.
- wc -l compte le nombre de lignes reçues.

## Partie II  : recherche

PS : nous n'avons pas vraiment abordé dans le cours, la commande find, nous le verrons un peu plus tard, cependant, à l'aide de find --help ou man find vous êtes en capacité de traiter cet exercice.

6. Fichiers modifiés dans les dernières 24h dans /etc.
```bash
find /etc -type f -mtime -1 
```
7. fichiers > 50 Mo dans /var/lib.
```bash
find /var/lib -size +50M -printf "Taille de %s Ko chemin %p\n"
```
8. Fichiers .log de plus d'1 jour avec les details
```bash
find / -name "*.log" -atime -1 -ls
```
- -name "*.log" : fichiers avec extension .log
- -mtime +1 : modifiés il y a plus d'un jour
- -ls : affiche les détails (droits, propriétaire, taille, date, etc.)
9. Fichiers de configuration SSH avec deux méthodes différentes.
```bash
find /etc/ssh/*/*.conf -type f
# ou bien
find /etc -name "*ssh*"
# ou bien
ls -lah /etc/ssh/*/*.conf
```

10. Création de plusieurs fichiers

touch /tmp/fichier{1..5}.tmp

11. Demande confirmation avant suppression de tous les fichiers dans /tmp crées aujourd'hui
```bash
find /tmp -ctime 0 -type f -ok rm {} \;
# sans demander la confirmation c'est l'option -exec qu'il faut utiliser
find /tmp -ctime 0 -type f -exec rm {} \;
```
- -ok demande confirmation avant l'exécution de la commande pour chaque occurence du résultat de find
- -exec s'exécute immédiatement pour tous les éléments du résultat
- {} est remplacé par le nom du fichier courant
- \; marque la fin de la commande exécutée.
- alternative à ctime
12. Via une commande sur une seule ligne, affichez la taille totale cumulée de tous les fichiers ayant l'extension .log
```bash
find / -type f -name "*.log" -exec du -cb {} + 2>/dev/null | tail -1
# ou bien
du -ch $(find /var/log -type f -name "*.log") 2>/dev/null | tail -1
```
- du -cb : affiche la taille de chaque fichier et un total en octets (-b).
- tail -1 : récupère uniquement la ligne du total qui se trouve tout en bas
- 2>/dev/null : optionnel, il masque les erreurs d'accès aux répertoires protégés. On l'utilise souvent pour masquer les erreurs aux utilisateurs
