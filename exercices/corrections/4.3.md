## Recap commandes

| Commande | Rôle                   |
| -------- | ---------------------- |
| `ln`     | lien physique          |
| `ln -s`  | lien symbolique        |
| `ls -li` | afficher inode         |
| `rm`     | supprimer              |
| `echo`   | écrire dans un fichier |
| `pwd`    | chemin courant         |


## Synthèse lien symbolique VS physique

|                            | Physique         | Symbolique |
| -------------------------- | ---------------- | ------------- |
| Inode partagé              | Oui              | Non           |
| Traverse FS                | Non              | Oui           |
| Casse si cible est supprimée | Non            | Oui           |
| Pointe vers                | inode            | chemin        |
| Peut cibler un répertoire  | Non | Oui           |

## Enoncé

1. Création fichier rapport.txt
```bash
echo "Version Initiale" > ~/rapport.txt
```
2. Création lien physique
```bash
ln rapport.txt lien_physique.txt
```
3. Création lien symbolique
```bash
ln -s rapport.txt lien_symbolique.txt
```
4. Affichage inodes et constats
```bash
ls -li
```
- rapport.txt et lien_physique.txt ont le même inode, le compteur de liens est à 2
- lien_symbolique.txt contient un inode différent et a un chemin vers le fichier cible rapport.txt
5. Modification du contenu et constats
```bash
echo "Modification via lien physique" > lien_physique.txt
cat rapport.txt
cat lien_symbolique.txt
```
- Le lien physique est une référence directe au même inode, donc modification affecte les mêmes blocs de données donc elle impacte le fichier d'origine (ce fichier est également modifié)
6. Supprission du fichier original `rapport.txt`.
```bash
rm rapport.txt
```
7. Lecture `lien_physique.txt` après suppression et constats
```bash
cat lien_physique.txt
```
- Le fichier existe
8. Lecture `lien_symbolique.txt` après suppression et constats
```bash
cat lien_symbolique.txt # lien cassé
```
9. Explications
| Type de lien  | Fonctionnement                      |
| ------------- | ----------------------------------- |
| Hard link     | pointe directement vers l’**inode**, marche même si le fichier d'origine est supprimé |
| Symbolic link | pointe vers un **chemin (texte)**, ne marche plus quand la cible est supprimée|

## Bonus

10. Création à nouveau du fichier rapport.txt
```bash
echo "Nouveau rapport" > rapport.txt
cat lien_symbolique.txt
```
- Le lien fonctionne à nouveau car le chemin est identique, cependant, il pointe vers un nouveau fichier différent (un autre inode)
11. Lien symbolique `mes_logs` qui cible `/var/log`
```bash
ln -s /var/log ~/mes_logs
cd ~/mes_logs
pwd
```
- On se trouve dans /home/<user>/mes_logs donc on reste dans le répertoire du dossier courant (lien symbolique) et non dans /var/log (la cible). Cependant, les fichiers cibles seront dans /var/log.
12. Lien symbolique vers /mnt/tutos et résultat
```bash
ln -s /mnt/tutos ~/tutos
```
- Tout fonctionne car le lien symbolique traverse les systèmes de fichiers.
13. Lien physique vers /mnt/tutos et résultat
```bash
ln /mnt/tutos/linux.txt ~/linux.txt # erreur
```
- Ne fonctionne pas car les liens physiques doivent être sur le même système de fichiers et donc ils ne peuvent pas traverser les partitions
- Un lien physique pointe vers le même inode et les inodes sont propres à une partition.
