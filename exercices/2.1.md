# Atelier 2.1 : partitionnement de disque et GRUB

## Objectifs

- [x] Ajouter un nouveau disque
- [x] Partitionner un disque
- [x] Personnaliser le comportement de GRUB

---

## Partie I : montage temporaire

1. Eteignez la VM debian
2. Dans les configurations, onglet stockage, ajoutez un deuxième disque dur virtuel VDI de 5Go avec une allocation dynamique et enregistrez
3. Démarrez la VM avec la nouvelle configuration
4. A l'aide de la commande lsblk, identifiez le nom attribué à votre nouveau disque.
```bash
lsblk
# PS : ça devrait être /dev/sdb
```
5. Utilisez fdisk ou gdisk sur ce nouveau disque pour créer une partition unique principale occupant toute l'espace
6. Formatez la nouvelle partition /dev/sdb1 avec le système ext4 à l'aide de la commande
```bash
mkfs.ext4 /dev/sdb1
```
7. Montez cette partition dans le dossier `/mnt/data`
8. Vérifiez que l'espace est disponible avec la commande suivante
```bash
df -h
```
9. Créez un nouveau dossier `test/`dans `/mnt/data`

---

## Partie II : montage persistante

10. Eteignez et redémarrez la machine
11. Exécutez les commandes suivantes, en observant le résultat, que pouvez-vous en déduire ?
```bash
df -h
ls -lah /mnt/data
```
12. Modifiez le fichier /etc/fstab avec nano et ajoutez la ligne suivante
```bash
/dev/sdb1 /mnt/data ext4 defaults 0 2
```
13. Testez la nouvelle configuration et effectuez votre montage à l'aide des commandes suivantes
```bash
fstab
df -h /dev/sdb1
sudo mount -a
```
14. Redémarrez à nouveau la machine
15. Exécutez à nouveau les commandes ci-après, que pouvez-vous en déduire ?
```bash
df -h
ls -lah /mnt/data
```
16. Démontez la partition /dev/sdb1 à l'aide la commande suivante, que pouvez-vous en déduire ?
```bash
sudo umount /dev/sdb1
```
17. Remontez la partition /dev/sdb1 dans /mnt/data, que pouvez-vous en déduire ?

---

## Partie III : GRUB

18. Ouvrez le fichier /etc/default/grub avec nano et modifiez la variable GRUB_TIMEOUT=2,  enregistrez et quittez nano
```bash
sudo nano /etc/default/grub
# Modification de cette ligne
GRUB_TIMEOUT=2
```
19. Que pouvez-vous déduire la manipulation précédente ?
20. Trouvez le moyen d'appliquer ces changements au prochain démarrage.
PS : les modifications effectuées ne sont pas pris en compte directement par GRUB au démarrage, il faut effectuer une commande qui va régénérer le vrai fichier de configuration de GRUB.
